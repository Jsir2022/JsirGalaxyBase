# JsirGalaxyBase 银行终端 GUI 设计

日期：2026-03-30

## 目标

银行不应该先做成“打开后再点一层空菜单”的纯导航页。

当前终端已经固定为：

- 左侧顶层导航
- 右侧主内容区
- 主内容区内部按板块展示
- 板块内再放二级跳转按钮

因此银行页也沿用同一个原则：

- 玩家一进入银行页，先看到关键内容
- 再通过内容区内的按钮进入二级子页
- 形成“顶层银行页 + 银行内部嵌套菜单”的结构

## 当前确认的信息架构

当前银行终端先按下面五页设计：

1. 银行主页
2. 个人账户页
3. 转账服务页
4. Exchange 公开页
5. 个人流水页

其中：

- 左侧顶层导航只放一个“银行”入口
- 银行内部子页不再占用左侧主导航
- 银行主页中直接放二级跳转按钮

## 银行主页要显示什么

银行主页不是空白目录，而是要先展示玩家最关心的内容。

当前确认首页至少显示下面几块：

### 1. 个人账户摘要

- 当前是否已开户
- 当前余额
- 账户状态
- 账户编号
- 最近一次账本更新时间

如果玩家还没开户，要明确显示：

- 当前未开户
- 银行系统不会自动开户
- 玩家必须先显式开户，后续转账不会替双方隐式建户

### 2. Exchange 公开摘要

- Exchange 储备账户当前余额
- 公开账户编号
- 最近一次账本更新时间

这块摘要的目的不是给管理员看，而是给普通玩家直接看到：

- 兑换储备当前还有多少能力
- 当前公开账本不是黑箱

### 3. 银行菜单板块

在首页主区直接放跳转按钮：

- 个人账户
- 转账服务
- Exchange 公开页
- 个人流水

### 4. 个人近期流水预览

- 展示玩家最近几条流水摘要
- 如果未开户或暂时没有流水，要明确写出来

### 5. 银行说明 / 当前阶段提示

- 当前阶段提供真实只读快照、当前玩家开户与玩家转账
- 任务书硬币兑换真实入口延期到市场阶段，本轮不接
- 银行页不会伪造贷款、理财、收益曲线等未实现功能

## 子页职责

### 1. 个人账户页

用途：

- 展示个人账户的完整状态信息
- 强化“是否开户、余额多少、账户编号是什么”

当前主要显示：

- 余额
- 账户状态
- 账户编号
- 币种
- 最近更新时刻

### 2. 转账服务页

用途：

- 先作为制度说明和后续正式操作入口预留页

当前主要显示：

- 玩家当前余额
- 转账前提条件
- 当前不自动开户
- 转账双方都必须已有账户
- 离线目标必须已开户，且服务端还能把玩家名解析回唯一 UUID
- 当前不收手续费
- 当前已经接入真实玩家转账提交
- 兑换相关真实提交入口明确延期到市场阶段

### 3. Exchange 公开页

用途：

- 把兑换储备账户的余额与近期账本公开给所有玩家

当前主要显示：

- Exchange 储备余额
- 账户编号
- 最近更新时间
- 最近几条 exchange 账本分录
- 公开透明说明

这页明确是玩家公开页，不是管理后台页。

### 4. 个人流水页

用途：

- 给玩家看自己最近的账本变化

当前主要显示：

- 最近几条个人流水
- 当前余额
- 若无账户或无流水时的明确提示

## 当前不显示的内容

当前明确不在银行终端里伪造下面这些东西：

- 贷款
- 利息
- 定存
- 理财产品
- 虚构的财富曲线
- 没有落地的数据图表

原因很直接：

- 当前银行一期底层真实能力是账户、流水、P2P 转账、系统内部划转基础、exchange 结算服务层 API 与公共储备记账
- 页面应该忠实反映当前制度实现，而不是做“金融皮肤”

## 交互原则

银行页当前按下面规则推进：

- 先做服务端权威只读快照
- 先稳定分页和嵌套菜单
- 在不越过阶段边界的前提下，只补开户和 P2P 转账这两个当前已定稿入口
- 兑换真实入口延后到市场阶段再接

因此当前阶段：

- 首页和子页可以是真实数据
- 转账页已经可以提交真实玩家转账
- 开户页已经可以提交当前玩家开户
- 兑换提交、市场能力和更复杂的确认链仍后置到市场阶段

## 本轮实施范围

本轮代码实现先覆盖：

- 左侧顶层新增“银行”入口
- 银行主页
- 个人账户页
- 转账服务页
- Exchange 公开页
- 个人流水页
- 个人账户与 Exchange 储备的真实只读快照接入

本轮暂不实现：

- GUI 内直接提交兑换
- 市场阶段的任务书硬币兑换真实入口
- 复杂筛选、分页器、搜索器
- 图表与图形化对账
